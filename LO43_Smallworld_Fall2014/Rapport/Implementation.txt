Après avoir généré les classes à partir de l'AGL, nous avons enfin pu rentrer dans le vif du sujet. Pour commencer, nous avons choisi de considérer le fait que nous aurions 3 joueurs pour ne pas avoir à se soucier des différents plateaux.
Ensuite, on a commencé à écrire les constructeurs et les méthodes principales. La classe Draw est presque complète et arrive a bien gérer la pioche. Nous avons aussi fait les écouteurs (listenners) sur le premières fenêtres (Demande du nombre de joueurs et l'entrée des noms des différents joueurs). 
Nous avons créer en parallèle les classes Turn et le système de choix de peuple. La classe turn permet de passer la main au joueur actif et de lui faire effectuer toutes les phases de jeu dans le bon ordre. La fenetre de choix du nouveaux peuple a tout d'abord été créé de manière simpliste avec un bouton pour chaque associationde peuple/pouvoir qui affiche l'or actuellement réservé a chaque peuple.
Nous avons décidé de créé le plateau a partir d'un fichier texte car il est plus facile de modifier le plateau de cette manière que si nous avions tout créer directement dans le code source du programme. Nous avions prévu de faire un plateau non rectangulaire et il fallait donc gérer un nombre indéfini de cases adjacentes 
Ensuite nous avons implémenté les phases de conquete et de redéploiement, qui intéragissent avec le plateau précédemment créé. Ces 2 étapes étant complexes nous avons dut rajoutés beaucoup de méthodes dans la majorité des classes de ce projet (par exemple le dé de renfort).
A cett étape du développement le joueur n'intéragit pas avec le plateau par la souris mais il selectionne des cases grace a leurs numéros quand l'invite de commande lui demande.
Apres cela il nous fallait encore compter les points des joueurs ainsi que determiner le vainqueur a la fin de la partie. C'est a ce moment la que nous avons commencé a gérer les pouvoirs dans notre jeu.
